Recursion: 5 key points

  • Define a base case (terminare).
  • Reduce the problem size at each call.
  • Assume recursive call is correct (induction intuition).
  • Beware of exponential branching (Fibonacci naive).
  • Complexity matters: memoization / DP for Fibonacci.

Template

func(n):
  if base(n): return ...
  return combine( func(smaller(n)), ... )

Common pitfalls

  • Missing base case → infinite recursion.
  • Base case too weak → wrong answers.
  • Huge recursion depth → stack overflow.
Q1. Complexity of naive recursive Fibonacci?
O(n)
O(2^n)
O(n log n)
Naive Fibonacci branches into two calls per level → exponential growth.
Q2. What must always exist in a recursive function?
A base case
A loop
Global variables
Without a base case, recursion does not terminate.
